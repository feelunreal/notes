Ниже приведен порядок развертывания проекта.

// mongo
Проект испольует в качестве хранилища - mongodb. Для простоты предполагается, что mongo запущена на том же хосте что и проект (на дефолтном для нее порту).
Коннект происходит без использования имени пользователя и пароля (опять же, чтобы упростить).

В корне проекта выполняем: npm i

Предполагается, что 3000 порт будет свободен в системе, иначе его можно выставить в конфиге приложения /config/config.json

Для запуска выполняем из корневой директории: node index.js
Если, вдруг, возникли проблемы с развертыванием, то работающее приложение доступно вот по этому адресу: http://178.63.42.89:3000

// подробнее о приложении и реализации:
Приложение содержит 2 страницы, страницу авторизации и главную страницу.
Реализовано REST API с доступом по токену, срок жизни токена: 24 часа (конфигурируется в конфиге).
При истечении жизни токена, пользовательская сессия заканчивается и пользователь должен заново залогиниться в приложение, после
чего ему будет выдан новый токен. Пользователь не может получить доступ через api к чужим заметкам. 
Синхронизация заметок сделана путем использования веб сокетов на socket.io. Логирование ведется в корень проекта в файл access.log.
